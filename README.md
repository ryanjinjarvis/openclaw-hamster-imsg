# Openclaw Hamster iMessage Extension

Production-ready scaffold for a TestFlight-bound iMessage extension that surfaces the latest "hamster" art pack with remote manifest sync, offline caching, and a searchable popularity-sorted grid UI.

## Project Layout

```
apps/openclaw-hamster-imsg/
├── HamsterIM.xcodeproj           # iOS host container + Messages extension target
├── HamsterIM/                    # Minimal host app wrapper
├── MessagesExtension/            # Sticker browsing UI + manifest client
├── engineering/                  # Contract + QA/build docs
├── index/                        # Upstream pipeline outputs (source of truth)
└── scripts/                      # Manifest export + validation helpers
```

## Requirements

- Xcode 15.4 or newer (iOS 17 SDK or higher) with the iMessage App Extension tooling installed.
- Apple Developer Program account with access to Certificates, Identifiers & Profiles.
- Node.js 18+ (already available in this workspace) for manifest export/validation scripts.

## Manifest lifecycle

1. **Source of truth** – `index/index.json` and related artifacts generated by the pipeline describe every scraped hamster asset.
2. **Export** – `scripts/export-approved-pack.js` filters the latest approved items, enforces the `engineering/DATA_CONTRACT_V1.json` data contract, and produces an app-ready manifest.
3. **Validation** – `scripts/validate-manifest.js` re-validates any manifest (including remote copies) before publishing.
4. **Delivery** – Ship the exported JSON to your CDN (URL configured in `MessagesExtension/Sources/Configuration/AppConfiguration.swift`).
5. **Runtime** – `ManifestRepository` fetches the remote manifest, validates it, caches the last known-good copy, and falls back to the bundled seed when offline.

### Export workflow

```bash
cd /Users/user123/.openclaw/workspace
node apps/openclaw-hamster-imsg/scripts/export-approved-pack.js \
  --input apps/openclaw-hamster-imsg/index/index.json \
  --output apps/openclaw-hamster-imsg/MessagesExtension/Resources/Manifest/exported-manifest.json \
  --limit 120 --minPopularity 10000 --seed true

node apps/openclaw-hamster-imsg/scripts/validate-manifest.js \
  apps/openclaw-hamster-imsg/MessagesExtension/Resources/Manifest/exported-manifest.json
```

- `--seed true` optionally replaces the bundled seed so the extension starts with the newest pack offline.
- Upload the exported manifest to your CDN and update `AppConfiguration.remoteManifestURL` if the location changes.

## Runtime behavior

- Search bar filters by tag, platform, or identifier.
- Grid is auto-sorted by popularity (ties broken by newest `createdAt`).
- Selecting a cell downloads the full-resolution image and inserts an `MSMessageTemplateLayout` into the active conversation.
- Manifest sync flows:
  - Load cached manifest if available.
  - Otherwise load bundled seed (`MessagesExtension/Resources/Manifest/hamster_manifest_seed.json`).
  - Kick off remote fetch; on success, validate per contract, cache to disk, and refresh UI; on failure, show offline toast and leave cached data intact.

## Build & archive

Detailed command-by-command steps live in `engineering/BUILD_AND_RELEASE.md`, including signing placeholders, but the TL;DR is:

1. Open `HamsterIM.xcodeproj` in Xcode.
2. Update the `HamsterIM` host app and `MessagesExtension` targets with your Team ID and bundle identifiers (e.g., `dev.openclaw.hamsterim` and `dev.openclaw.hamsterim.MessagesExtension`).
3. Provide production app icons, set the App Group if you decide to share data with other extensions, and configure provisioning profiles.
4. Use `Product > Archive`, then distribute via `Organizer > Distribute App > App Store Connect > Upload`.
